Programación y Desarrollo

1.SINTAXIS:

Sintaxis es el conjunto de reglas que definen cómo debe escribirse correctamente el código en un lenguaje de programación. Como en un idioma que usamos normalmente hay normas gramaticales en los lenguajes de programación existen normas en el orden de los símbolos que deben respetarse. Una mala sintaxis provoca errores de compilación o ejecución, haciendo que haya un fallo al correr el sistema. Por ejemplo, en Python es obligatorio usar dos puntos : al final de una condición if, mientras que en Java se usan llaves para delimitar los bloques de código.

2.IDENTACIÓN:

La identación se refiere al uso de espacios o tabulaciones al inicio de las líneas de código con el fin de organizar visualmente la estructura de un programa. Aunque en muchos lenguajes es opcional, en otros como Python es fundamental, ya que marca los bloques de instrucciones y su jerarquía. Una buena identación mejora la legibilidad del código, facilita el trabajo en equipo y ayuda a prevenir errores lógicos, además de ser considerada una buena práctica de programación.

3.Script:

Un script es un programa pequeño compuesto por una secuencia de instrucciones diseñadas para automatizar tareas sencillas o repetitivas. Se diferencia de un software completo porque generalmente no requiere compilación y se ejecuta mediante un intérprete. Ejemplos comunes incluyen scripts en Bash para automatizar procesos en Linux o scripts en Python para renombrar archivos, enviar correos o procesar datos de forma automática.

4.Código:

El código es el conjunto de instrucciones escritas en un lenguaje de programación que indican a la computadora qué acciones realizar. Puede escribirse en diferentes niveles de abstracción: alto nivel (como Java, Python o C#), más cercano al lenguaje humano, o bajo nivel (ensamblador), más próximo al lenguaje de máquina. El código fuente debe seguir la sintaxis del lenguaje, y luego ser interpretado o compilado para transformarse en un programa ejecutable.

5.Código binario:

El código binario es el sistema de representación que utiliza únicamente los números 0 y 1 para almacenar y procesar información en una computadora. Cada 0 o 1 se denomina bit, y en conjunto forman instrucciones o datos que la máquina puede comprender directamente. Este sistema es el lenguaje nativo de los procesadores, y todo software escrito en lenguajes de alto nivel termina convertido en código binario para ser ejecutado.

6.ASCII:

ASCII (American Standard Code for Information Interchange) es un estándar de codificación de caracteres que asigna valores numéricos a letras, números, símbolos y caracteres de control. Por ejemplo, la letra "A" se representa con el número 65 en ASCII. Este sistema permitió la interoperabilidad entre diferentes computadoras y dispositivos en los primeros años de la informática, y aún hoy sigue siendo ampliamente utilizado en sistemas de comunicación y almacenamiento de datos.

7.Fetch:

Fetch es la primera etapa del ciclo de instrucción de una CPU. En esta fase, el procesador obtiene (o “trae”) la instrucción que debe ejecutar desde la memoria principal. Es un paso fundamental, ya que prepara al sistema para interpretar la instrucción y luego procesarla. Sin esta etapa, el procesador no sabría qué operación debe realizar, convirtiéndose en un “cerebro sin órdenes”.

8.Decode:

Es la decodificación que ocurre justo después del fetch. En ella, la CPU interpreta la instrucción obtenida de la memoria, identificando qué operación debe realizar como una suma, una comparación o una transferencia de datos esta fase convierte la instrucción en señales de control que guían a las demás unidades del procesador es clave porque transforma un simple número binario en una acción específica.

9.Execute:

En programación, execute significa ejecutar que es un conjunto de instrucciones que forman parte de un programa o script. Cuando se ejecuta un código el traductor toma las órdenes escritas por el programador y las convierte en acciones reales que el computador entiende y procesa e interpreta línea por línea y realiza las operaciones solicitadas como mostrar un mensaje hacer cálulos o comunicarse con una base de datos.

10.Write-back:

En arquitectura de computadores write-back es una técnica de memoria caché en la que los datos modificados por el procesador se actualizan primero en la caché y solo se escriben en la memoria principal cuando es estrictamente necesario, por ejemplo, cuando ese bloque de caché debe ser reemplazado esto mejora el rendimiento porque evita escribir en la memoria principal con cada cambio reduciendo la carga hacia la RAM pero se requiere mecanismos adicionales de coherencia para asegurarse de que los datos en caché y en memoria principal no entren en conflicto.

11.parametro

Un parámetro es un valor que se le pasa a una función o procedimiento para que pueda trabajar con datos específicos sin tener que estar escribiendo valores directamente en el codigo fuente es como pensar en una función como si fuera una máquina: el parámetro es la pieza o dato que colocas para que la máquina de un resultado diferente cada vez esto te permite reutilizar el mismo bloque de código con distintos valores de entrada.

12.Endpoint

Es una puerta de entrada en una API que normalmente es representada por una URL a través de la cual los clientes pueden acceder a ciertos recursos o funcionalidades. Por ejemplo, si tienes una API de usuarios, un endpoint podría ser /usuarios y otro /usuarios/{id} para obtener datos específicos. Es el punto de contacto entre el cliente y el servidor.

13.FastAPI

FastAPI es un framework moderno y rápido de Python diseñado para crear APIs de manera sencilla, eficiente y con un alto rendimiento. Utiliza tipado estático para generar documentación automática y aprovechar las ventajas de validación de datos que es usado en proyectos que necesitan rapidez y escalabilidad, como microservicios o aplicaciones que necesitan comunicación en tiempo real.

14.API

Una API (Application Programming Interface) es un conjunto de reglas y herramientas que permiten a diferentes programas comunicarse entre sí. En vez de preocuparte por cómo está hecha una aplicación por dentro la API expone ciertas funciones o datos que puedes consumir Es como un “menú” en un restaurante no es necesario saber cómo se cocina solo pides del menu.

15.REST

REST (Representational State Transfer) es un estilo de arquitectura para diseñar APIs que funciona sobre HTTP. Sus principios son trabajar con recursos (usuarios, productos, pedidos), usar métodos estándar (GET, POST, PUT, DELETE) y ser sin estado cada petición es independiente esto lo hace simple, escalable y fácil de entender.

16.SOAP

SOAP (Simple Object Access Protocol) es un protocolo de mensajería basado en XML que se utiliza para la comunicación entre aplicaciones y en vez de REST, SOAP es más estricto y requiere seguir reglas muy definidas lo cual lo hace más seguro y confiable en entornos empresariales aunque también más pesado y complejo.

17.RPC

RPC (Remote Procedure Call) es un mecanismo que permite a un programa ejecutar una función o procedimiento que está en otro servidor como si fuera local y el cliente solo llama al procedimiento remoto y no se preocupa de cómo se maneja la comunicación en segundo plano esto hace más simple el desarrollo distribuido aunque puede generar dependencia fuerte entre servicios.

18.WebSocket

Un WebSocket es un protocolo que permite comunicación en tiempo real entre cliente y servidor con diferencia de HTTP, que funciona con peticiones y respuestas WebSocket mantiene una conexión abierta que ideal para chats, juegos online o aplicaciones de bolsa donde la información cambia constantemente.

19.Algoritmo

Un algoritmo es una serie de pasos lógicos y ordenados que permiten resolver un problema o realizar una tarea. Es como una receta de cocina: tienes los ingredientes, un proceso de cocina y el plato. En programación, los algoritmos son la base de cómo resolvemos problemas de forma eficiente.

20.Variable

Una variable es un espacio en memoria con un nombre que usamos para guardar un valor que puede cambiar durante la ejecución del programa es como una caja etiquetada puedes abrirla ver qué contiene, cambiar lo que hay dentro y volver a usarla más tarde. Sin variables, el programa no podría manejar datos dinámicos.

21.constante

Una constante es un espacio en memoria que almacena un valor fijo que no puede cambiar durante la ejecución del programa este sirve para guardar datos que son importantes y estables, como el valor de π (3.1416) o el número de días en una semana usar constantes ayuda a mantener el código más legible y seguro evitando modificaciones accidentales.

22.FUNCION

Una función es un bloque de código que realiza una tarea específica y puede ser reutilizado en diferentes partes de un programa lo más común es que recibe parámetros, ejecuta un proceso y devuelve un resultado. Es como una máquina con entrada y salida se le dan datos, los procesa y te entrega un resultado.

23.PROCEDIMIENTO

Un procedimiento es parecido a una función solo que normalmente no devuelve un valor, sino que ejecuta una acción como decir un procedimiento podría imprimir un mensaje en pantalla o guardar datos en una base. Es útil cuando quieres organizar el código en acciones reutilizables.

24.CICLO

Un ciclo es una estructura que permite repetir un bloque de código varias veces mientras se cumpla una condición existen ciclos como for (cuando sabes cuántas veces repetir) y while (cuando repites hasta que algo cambie). Son la base de la automatización en programación.

25.ITERACION

Una iteración es cada repetición que ocurre dentro de un ciclo. Si un ciclo se ejecuta 10 veces, entonces hay 10 iteraciones. Cada iteración representa un paso del proceso repetitivo.

26.CONDICIONAL

Una condicional es una estructura que permite ejecutar diferentes bloques de código dependiendo de si se cumple o no una condición lógica. Por ejemplo, si un número es mayor a 18, entonces se ejecuta una acción, y si no, otra. Esto da flexibilidad al programa para tomar decisiones.

27.BOOLEANO

Un booleano es un tipo de dato que solo puede tener dos valores: verdadero (O) o falso (X). Es fundamental en programación porque permite evaluar condiciones lógicas y controlar el flujo del programa.

28.COMPILADOR

Un compilador es un programa que traduce todo el código fuente escrito por el programador en un lenguaje de alto nivel a lenguaje máquina antes de ejecutarlo. Esto genera un archivo ejecutable que puede correr directamente en el sistema operativo

29.INTERPRETE

Un intérprete es un programa que lee y ejecuta el código fuente línea por línea sin necesidad de compilarlo previamente. Esto facilita la depuración y pruebas rápidas, pero generalmente es más lento que un compilador

30.LENGUAJE DE PROGRAMACION

Un lenguaje de programación es un sistema de reglas, símbolos y palabras clave que permiten a los desarrolladores dar instrucciones a la computadora. Puede ser de bajo nivel que está más cercano a la máquina que al lenguaje que usamos las personas como también se usa el de alto nivel que se acerca más al lenguaje que las personas usamos.

31.LENGUAJE DE BAJO NIVEL

Son lenguajes muy cercanos al lenguaje máquina, como el ensamblador. Permiten un control muy detallado sobre el hardware, pero son más difíciles de aprender y escribir.

32.LENGUAJE DE ALTO NIVEL

Son lenguajes más fáciles de leer y escribir para los humanos, como Python o Java. Abstraen los detalles del hardware para que el programador se concentre en la lógica del problema.

33.pseudocodigo

El pseudocódigo es una forma de escribir algoritmos utilizando un lenguaje intermedio entre el lenguaje natural y la programación. No lo entiende la computadora, pero es muy útil para planificar soluciones antes de codificarlas.

34.DIAGRAMA DE FLUJO

Un diagrama de flujo es una representación gráfica de un algoritmo o proceso mediante símbolos y flechas permite visualizar la lógica paso a paso y facilita la comprensión de cómo fluye la información.

35.DEPURACION

La depuración es el proceso de detectar, analizar y corregir errores en un programa involucra revisar el código, probarlo y aplicar técnicas que ayuden a entender dónde y por qué ocurre el fallo.

36.BUG

Un bug es un error o defecto en un programa que provoca que ocurra un fallo o que ocurra una reacción inesperada puede ser desde un pequeño fallo visual hasta un error crítico que detenga la ejecución del sistema.

37.REPOSITORIO

Un repositorio es un espacio donde se almacena y gestiona el código de un proyecto, generalmente en plataformas como GitHub o GitLab. Allí se guarda el historial de cambios, versiones y ramas del proyecto.

38.CONTROL DE VERSIONES

El control de versiones es un sistema que permite registrar los cambios realizados en el código a lo largo del tiempo. Ayuda a colaborar en equipo, volver a versiones anteriores y organizar el desarrollo.

39.COMMIT

40.BRANCH

Branch o rama es una copia del código principal donde puedes hacer modificaciones sin afectar la versión estable se usa para desarrollar nuevas funciones o corregir errores de forma aislada y luego se puede usar con el código principal.

41.MERGE(FUSION)

42.PULL REQUEST(PR)

43.FORK

44.CLONE

45.PUSH

46.PULL

47.TAG (ETIQUETA)

48.RELEASE(LANZAMIENTO)
